# 考勤自动保存功能 — 手动测试指南

## 前置条件

1. 确保 PHP 开发服务器已运行（例如 `php -S localhost:8080`）
2. 确保已运行测试数据种子脚本：`php tests/seed_test_data.php`
3. 使用测试账号登录：用户名 `testprof`，密码 `test1234`
4. 进入测试班级页面（代码：`TEST-2026`）

---

## 测试步骤

### 测试 1：选择学生 → 出现橙色圆点

1. 打开班级页面
2. 在考勤区域，点击 5 个学生的按钮（按钮会高亮显示）
3. **预期结果**：在"考勤"标题旁边出现一个 **橙色圆点**（#f59e0b），表示有待同步的数据

### 测试 2：等待 3 秒 → 圆点变为绿色

1. 停止操作，等待约 3 秒钟（空闲超时时间为 2.5 秒）
2. **预期结果**：橙色圆点变为 **绿色圆点**（#10b981），表示数据已成功同步到服务器
3. 刷新页面，之前选中的 5 个学生应该仍然处于选中状态

### 测试 3：切换到其他标签页 → 立即同步

1. 再次点击几个学生（确保出现橙色圆点）
2. 切换到浏览器的另一个标签页（或最小化窗口）
3. 等待 1 秒后切回来
4. **预期结果**：圆点应该已经变为 **绿色**，因为在页面隐藏时触发了立即同步

### 测试 4：关闭并重新打开页面 → 恢复之前的考勤

1. 选择 3-5 个学生，等待同步完成（绿色圆点）
2. 完全关闭浏览器标签页
3. 重新打开班级页面
4. **预期结果**：之前选中的学生仍然处于选中状态，数据从服务器恢复

### 测试 5：断开网络 → 红色圆点 + 重试按钮

1. 断开网络连接（可以在浏览器开发者工具中设置为"离线"模式，Network → Offline）
2. 点击几个学生
3. 等待 3 秒
4. **预期结果**：
   - 圆点变为 **红色**（#ef4444），表示同步失败
   - 出现一个 **"Reintentar"**（重试）按钮
5. 点击"Reintentar"按钮 → 仍然显示红色（因为网络未恢复）

### 测试 6：恢复网络 → 点击重试 → 绿色圆点

1. 恢复网络连接（Network → Online）
2. 点击页面上出现的 **"Reintentar"** 按钮
3. **预期结果**：圆点变为 **绿色**，数据成功同步

## 状态指示器说明

| 颜色 | 状态 | 说明 |
|------|------|------|
| 🟠 橙色 (#f59e0b) | 待同步 | 有本地修改等待发送到服务器 |
| 🟢 绿色 (#10b981) | 已同步 | 数据已成功保存到服务器 |
| 🔴 红色 (#ef4444) | 同步失败 | 网络错误，需要手动重试 |

## 自动同步触发条件

1. **空闲超时**：停止操作 2.5 秒后自动同步
2. **页面隐藏**：切换标签页或最小化时立即同步
3. **页面加载**：如果有未同步的本地数据，立即发送
4. **手动重试**：点击"Reintentar"按钮

## 数据清理

- 本地存储最多保留 7 天的考勤数据
- 超过 7 天的数据会在下次页面加载时自动清理
- 成功同步的数据会被标记为"已同步"
